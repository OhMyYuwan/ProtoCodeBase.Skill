# Code of Conduct

ProtoCodeBase projects should be useful, respectful, and precise.

## Expected Behavior

- Be kind and direct.
- Assume good intent while still naming real problems.
- Keep technical disagreement focused on evidence and tradeoffs.
- Respect project boundaries, licensing boundaries, and private data.
- Help contributors understand why a change is requested.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language
- Publishing private information without permission
- Submitting secrets, credentials, or user data
- Intentionally misleading users about license or protocol behavior
- Using issues or pull requests for spam

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; reject pull
requests; or block participants whose behavior harms the project.
